## GitHub ve Kurulum Haritası

Bu dosya, bu projeyi başka kullanıcıların kendi bilgisayarlarında ve kendi GitHub hesaplarında nasıl kullanacağını adım adım anlatır.

---

### 1. Gereksinimler

- **Docker**: 20.10 veya üzeri
- **Git**: 2.30 veya üzeri
- Host makinede ROS 2 kurulumu **gerekmiyor** (her şey Docker konteyneri içinde çalışıyor).

---

### 2. Projeyi Klonlama ve Çalıştırma (Başka Kullanıcılar İçin)

Bu adımlar, başka bir kullanıcının bu repoyu kendi bilgisayarında çalıştırması içindir.

1. Depoyu klonlayın:

```bash
git clone https://github.com/<kullanici-adi>/<repo-adi>.git
cd <repo-adi>
```

2. Docker imajını oluşturun:

```bash
docker build -t myrosapp .
```

3. Konteyneri çalıştırın:

```bash
docker run --rm myrosapp
```

4. Ayrı bir terminalde çalışan konteynere girin:

```bash
docker ps                  # container_id'yi not edin
docker exec -it <container_id> bash
```

5. Konteyner içinde doğrulama komutları:

```bash
ros2 topic list
ros2 topic echo /processed_value
ros2 service call /compute_command "input: 12.5"
```

---

### 3. Bu Projeyi Kendi GitHub Hesabınıza Taşımak (Terminal ile)

Bu adımlar, bu projeyi **kendi GitHub hesabında** yeni bir repo olarak yayınlamak içindir.

1. Orijinal projeyi klonlayın:

```bash
git clone https://github.com/<orijinal-sahip>/<orijinal-repo>.git
cd <orijinal-repo>
```

2. Mevcut `origin` remote'unu kaldırın:

```bash
git remote remove origin
```

3. Kendi GitHub hesabınızda **boş** bir repo oluşturun  
   Örnek isim: `myrosapp-ros2-humble`

4. Yeni remote'u ekleyin:

```bash
git remote add origin https://github.com/<sizin-kullanici-adiniz>/myrosapp-ros2-humble.git
```

5. Kodları kendi reponuza gönderin:

```bash
git branch -M main
git push -u origin main
```

Bu adımlardan sonra proje artık sizin GitHub hesabınızda barınır ve diğer kullanıcılar da sizin URL’nizle klonlayabilir.
