## YouTube Video Çekim Haritası (Senaryo)

Bu dosya, ödev için istenen videoyu adım adım nasıl çekeceğini anlatır.

### 1. Hazırlık

- **Görüntü ayarları**
  - Ekran kaydı + webcam (yüzün görünecek).
  - En az iki terminal penceresi açık olsun.
- **Proje klasörüne geç**

```bash
cd "/home/selamicetin/Masaüstü/Robotik Dersi Odev-3"
```

### 2. Videonun Akışı

#### A) Giriş

1. Yüzün görünürken kısa tanıtım yap:
   - İsmini söyle.
   - "Bu videoda Docker içinde çalışan 3 düğümlü ROS 2 Humble projemi göstereceğim." de.

#### B) `ssf.sh` ve `SSF_HASH.txt`

2. Proje klasörünü göster:

```bash
ls
```

3. `ssf.sh` script’ini çalıştır ve `SSF_HASH.txt` içeriğini göster:

```bash
./ssf.sh
cat SSF_HASH.txt
```

#### C) Docker imajını gösterme / oluşturma

4. Eğer imaj önceden varsa:

```bash
sudo docker images | grep myrosapp
```

5. Yoksa bu adımda oluştur:

```bash
sudo docker build --no-cache -t myrosapp .
sudo docker images | grep myrosapp
```

#### D) Konteyneri çalıştırma

6. Aynı terminalde konteyneri çalıştır:

```bash
sudo docker run --rm --name rosapp myrosapp
```

- Bu terminali açık bırak; düğümlerin logları görünsün.

#### E) İkinci terminal: konteynere girme

7. Yeni terminal aç, çalışan konteyneri bul:

```bash
sudo docker ps
```

8. `CONTAINER ID`'yi not et ve konteynere gir:

```bash
sudo docker exec -it rosapp bash
```

#### F) Doğrulama komutları (konteyner içinde)

9. Topic listesini göster:

```bash
ros2 topic list
```

10. İşlenmiş veriyi izle:

```bash
ros2 topic echo /processed_value
```

- Birkaç saniye bekle, sayıların aktığını göster.

11. Servisi çağır (doğru servis tipiyle):

```bash
ros2 service call /compute_command command_server_interfaces/srv/ComputeCommand "{input: 12.5}"
```

- Çıktıda `"output: HIGH"` yazdığını göster.

#### G) Kapanış

12. `Ctrl+C` ile `ros2 topic echo` ve gerekirse konteyneri durdur.
13. Yüzün görünürken kısa özet yap; hem sistemin çalıştığını söyle hem de bileşenleri kısaca açıkla:
    - **sensor_publisher**: 0.1 s’de bir `random.uniform(0, 20)` sensör verisini `/sensor_value` topiğine gönderir.
    - **data_processor**: `/sensor_value` dinleyip veriyi 2 ile çarpar ve `/processed_value` yayınlar; bu sayede temel bir işleme zinciri gösterilir.
    - **command_server**: `/compute_command` servisinde gelen `input > 10` ise `"HIGH"`, değilse `"LOW"` yanıtlar; böylece işlenmiş değere göre karar çıktısı üretilir.
    - Özet cümle: "3 düğüm (yayıncı, işleyici, servis), 2 topic ve 1 servis Docker içinde çalışıyor; hepsinin testlerini konteyner içinden yaptım."
14. Kaydı durdur ve videoyu YouTube’a **Unlisted** olarak yükle.
